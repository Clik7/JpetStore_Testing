Ok, l’errore è chiaro 👍
Il problema non è il certificato, ma come è configurato il proxy (è un errore molto comune con JMeter).
Andiamo per gradi.

1️⃣ Errore principale nella configurazione del proxy
Nel browser hai impostato:


Indirizzo proxy: http://localhost ❌


Porta: 8888 ✔️


👉 Questo è sbagliato
Nel campo indirizzo proxy NON va messo http://, ma solo l’host.
✅ Configurazione corretta
Indirizzo proxy: localhost
Porta: 8888

oppure (ancora meglio):
Indirizzo proxy: 127.0.0.1
Porta: 8888

📌 http:// serve negli URL, non nelle impostazioni proxy.

2️⃣ Verifica che il Proxy di JMeter sia realmente avviato
In JMeter devi avere:


Test Plan


Thread Group


HTTP(S) Test Script Recorder




Nel recorder:


Porta: 8888


Target Controller: impostato (es. Recording Controller)


Poi devi cliccare START (icona ▶️ verde).
👉 Se il recorder non è avviato, il browser darà ERR_PROXY_CONNECTION_FAILED.

3️⃣ Browser: controlli fondamentali
Chrome / Edge


Proxy manuale


HTTP proxy → localhost / 127.0.0.1


Porta → 8888


Usa lo stesso proxy per HTTPS ✔️


Firefox (consigliato)
Firefox è molto più semplice con JMeter.
Impostazioni → Rete:


Configurazione manuale proxy


HTTP Proxy: 127.0.0.1


Porta: 8888


✔️ Usa questo proxy anche per HTTPS



4️⃣ Certificato HTTPS (già fatto bene 👍)
Hai fatto correttamente se:


Hai esportato ApacheJMeterTemporaryRootCA.crt


L’hai importato in Autorità di certificazione attendibili


Hai riavviato il browser


⚠️ Se usi Firefox, il certificato va importato anche dentro Firefox, non solo Windows.

5️⃣ Firewall / Antivirus
Se ancora non va:


Disattiva temporaneamente antivirus / firewall


Alcuni bloccano la porta 8888


Puoi anche provare una porta diversa:
8889
8899

e aggiornarla sia in JMeter che nel browser.

6️⃣ Test rapido per capire se il proxy risponde
Con proxy attivo:


Apri browser


Vai su: http://example.com


Se:


❌ Errore immediato → proxy non in ascolto


✅ Vedi richieste comparire in JMeter → funziona



✅ Riepilogo veloce (checklist)
✔️ Proxy browser: localhost / 127.0.0.1 (senza http://)
✔️ Porta uguale in browser e JMeter
✔️ Recorder avviato (▶️)
✔️ Certificato importato
✔️ Browser riavviato

Se vuoi, dimmi:


che browser stai usando


se JMeter è Windows / Mac / Linux


se l’errore compare subito o solo su HTTPS


e ti guido passo-passo fino a vedere le richieste dentro JMeter 🔍🚀
